# NTB Seychelles — Procurement Dashboard

[![Deploy Dashboard](https://github.com/YOUR_USERNAME/ntb-seychelles/actions/workflows/deploy.yml/badge.svg)](https://github.com/YOUR_USERNAME/ntb-seychelles/actions/workflows/deploy.yml)

**Live dashboard → [YOUR_USERNAME.github.io/ntb-seychelles](https://YOUR_USERNAME.github.io/ntb-seychelles)**

An open-source, fully static procurement intelligence dashboard combining all four public data sources published by the [National Tender Board of Seychelles](https://www.ntb.sc). No server required — runs entirely in the browser via GitHub Pages.

---

## Architecture

```
Scrapers (Python) → data/*.csv → build.py → docs/index.html → GitHub Pages
```

| Layer | Technology | Where it runs |
|---|---|---|
| Scraping | Python · requests · BeautifulSoup · pdfplumber | Locally (your machine) |
| Data storage | CSV files committed to `data/` | GitHub repo |
| Dashboard build | `build.py` — embeds CSV data as JSON into HTML | GitHub Actions / locally |
| Dashboard serving | Pure HTML + Plotly.js | GitHub Pages (browser only) |

No database, no backend, no server costs, no runtime Python.

---

## Dashboard sections

- **Time range badge** — shows the date span covered by the current data
- **KPI strip** — total SR awarded, tender openings, EOI count, advertised count
- **Awarded spend** — SR value by category and by organisation
- **Top 10 winning bidders** — ranked by total SR value received
- **Bunching analysis** — scatter plot, one dot per contract, showing clustering near round-number thresholds
- **Bidding market** — bid count distribution + most frequent first-position bidders
- **Organisation deep dive** — per-entity breakdown across all four stages
- **Download** — export any dataset as CSV or JSON directly from the browser

## Quickstart

### 1. Fork & clone

```bash
git clone https://github.com/YOUR_USERNAME/ntb-seychelles.git
cd ntb-seychelles
pip install -r requirements.txt
```

### 2. Scrape the data

```bash
# All four scrapers (takes ~60–90 min for full runs)
python scrapers/ntb_seychelles_scraper.py --out data/ntb_tenders.csv
python scrapers/ntb_minutes_scraper.py    --out data/ntb_minutes.csv
python scrapers/ntb_eoi_scraper.py        --out data/ntb_eoi.csv
python scrapers/ntb_advertised_scraper.py --out data/ntb_advertised.csv

# Quick test run (3 pages each, ~2 min)
python scrapers/ntb_seychelles_scraper.py --out data/ntb_tenders.csv --max-pages 3
```

### 3. Build the dashboard

```bash
python build.py
# → writes docs/index.html

# Open locally to verify
open docs/index.html          # macOS
xdg-open docs/index.html      # Linux
start docs/index.html         # Windows
```

### 4. Deploy to GitHub Pages

```bash
git add data/*.csv docs/index.html
git commit -m "Initial data + dashboard"
git push
```

Then in GitHub: **Settings → Pages → Source → Deploy from branch → main → /docs → Save**

Your dashboard is live at `https://YOUR_USERNAME.github.io/ntb-seychelles/`

## Keeping data fresh

Option A — push CSVs manually:
```bash
python scrapers/ntb_seychelles_scraper.py --out data/ntb_tenders.csv
# ... (other scrapers)
python build.py
git add data/*.csv docs/index.html
git commit -m "Refresh $(date +%Y-%m-%d)"
git push
```

Option B — GitHub Actions does it automatically. The workflow in `.github/workflows/deploy.yml` triggers on every push to `main` that touches any `data/*.csv` file and rebuilds `docs/index.html` automatically.

## Data sources

All data is sourced from public NTB web pages. Accessible periods:

| Dataset | Pages | Notes |
|---|---|---|
| Awarded tenders | PDF-based, 4 periods accessible | Older PDFs return 404 |
| Minutes of tenders | ~158 listing pages | PDFs mostly accessible |
| Expressions of interest | ~28 listing pages | Data in HTML, PDFs mostly 404 |
| Advertised tenders | ~153 listing pages | PDFs accessible, richest data |

## Contributing

Pull requests welcome. To add a new data source:
1. Add a scraper in `scrapers/`
2. Add a loader function in `build.py`
3. Add a chart section to the `HTML_TEMPLATE` in `build.py`
4. Update `data/README.md`
